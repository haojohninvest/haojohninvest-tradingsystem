## 二、發現的問題詳情

### 問題 1：週末髒資料（最嚴重）

- **總筆數**：43,399 筆
- **時間範圍**：2016-01-30 ~ 2025-08-23
- **特徵**：
  - 大部分是「前一個交易日的複製」（如 2024/06/15 100% 與 06/14 相同）
  - 少數是異常值（如 2025/07/20 的 1058 筆股票有價格）
- **根因**：`run_crawler.py` 的 `--date` 分支沒有檢查 `weekday() >= 5`，手動指定週末時也會執行

### 問題 2：平日資料錯誤（影響更大）

**抽樣比對（DB vs 證交所 API）**：

| 日期 | 股票 | DB 收盤價 | API 正確價 | 差異 |
|------|------|-----------|-----------|------|
| 2024/5/20 | 1101 台泥 | 25.50 | **應為 33.90** | -8.40 |
| 2024/5/20 | 2330 台積電 | 1,055.00 | **應為 835.00** | +220 |
| 2024/5/20 | 1519 華城 | 564.00 | **應為 741.00** | -177 |
| 2024/5/20 | 2404 漢唐 | 611.00 | **應為 353.00** | +258 |

**結論**：2024/5/20 是「除權息除息日」，證交所 API 的 CSV 欄位順序會變動，爬蟲沒有正確處理，導致「收盤收盤價」欄位抓到錯誤的值。

### 問題 3：特定異常值散佈

| 異常收盤價 | 筆數 | 推測原因 |
|-----------|------|---------|
| 19.30 | 4,426 筆 | 某些股票的「漲跌」欄位被誤認為「收盤價」 |
| 100 | 3,549 筆 | 同上，或其他欄位偏移 |
| 757 | 40 筆 | 2025/07/20 週日的異常集中值 |

## 三、錯誤根因分析

### 根因 A：`run_crawler.py` --date 未檢查週末

```python
# 目前有問題的 code（line 31-34）
elif options.get('date'):
    target_date = datetime.strptime(options['date'], '%Y-%m-%d').date()
    self.stdout.write(f"正在處理 {target_date}...")
    MarketCrawler.run_daily_crawl(target_date)  # 週末也跑！
```

**影響**：手動執行 `--date 2025-07-20`（週日）或每次 Weekly Scheduler 的 `start_date=2020-01-01` 迴圈觸發到這些週末日期時，就會產生髒資料。

### 根因 B：證交所 API 欄位變動（CSV 解析脆弱）

爬蟲 `crawler.py` 的解析邏輯：
```python
# TWSE：直接使用中文欄位名稱定位
df.rename(columns={'證券代號': 'code', '證券名稱': 'name', ...})
```

**問題**：在除權息日，證交所 CSV 會多一個「除權息參考價」欄位，導致後續所有欄位**向右偏移一格**，爬蟲抓到的「收盤價」實際上是「漲跌幅」或其他欄位。

### 根因 C：沒有資料驗證機制

- 爬蟲寫入前沒有驗證「價格是否合理」（例如漲跌幅超過 limit）
- 沒有與前一日收盤價比對，確認是否異常跳動
- 沒有與「是否為交易日」交叉檢查

## 四、改善方案

### Phase 1：緊急止血（1~2 小時）

1. **停止 Scheduler**（已完成 detect_bad_dates 終止）
2. **刪除全部週末資料**（43,399 筆）
   ```sql
   DELETE FROM market_data_dailyprice WHERE strftime('%w', date) IN ('0', '6');
   ```
3. **修正 `run_crawler.py` 增加週末檢查**（10 分鐘修改）
   ```python
   elif options.get('date'):
       target_date = datetime.strptime(options['date'], '%Y-%m-%d').date()
       if target_date.weekday() >= 5:
           self.stdout.write(self.style.WARNING(f"{target_date} 是週末，跳過爬取"))
           return
       ...
   ```

### Phase 2：爬蟲加固（1~2 天）

1. **使用欄位索引而非中文欄位名稱定位**（防範除權息日欄位偏移）
   ```python
   # 用索引而非名稱
df = pd.read_csv(StringIO(csv_data), header=None, skiprows=1)
   df.columns = ['code', 'name', 'volume', 'trade_value', 'open', 'high', 'low', 'close', ...]
   ```
2. **增加價格合理性驗證**
   - 收盤價不得為負
   - 收盤價不得為 0
   - 收盤價與前一日比較，漲跌幅不得超過 ±50%（極端異常）
3. **增加週末交易日查詢**
   - 先查證交所「是否為交易日」API，非交易日不爬

### Phase 3：資料重建（1~2 週）

**目標**：重建 2020-01-01 ~ 2026-05-20 的乾淨資料
