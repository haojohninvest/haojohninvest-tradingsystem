# 自動排程服務設定說明

## 📋 功能說明

自動排程服務會在指定時間自動執行以下任務：

### 每日任務（週一到週五 13:30）
1. **爬取當日股價** - 從 TWSE 和 OTC 爬取當日交易資料
2. **計算技術指標** - 計算所有股票的 EMA、SMA、漲跌率等
3. **計算族群背離** - 計算各族群的 EMA20 乖離率與訊號

### 每週任務（週日 01:00）
1. **重新爬取所有歷史資料** - 從 2020-01-01 重新爬取
2. **重新計算所有指標** - 確保指標資料正確
3. **重新計算族群背離** - 更新所有歷史背離資料

---

## 🚀 啟動方式

### 方法一：手動啟動（測試用）

在終端機執行：
```bash
python manage.py start_scheduler
```

### 方法二：使用啟動腳本（推薦）

**Windows 10/11：**

1. 雙擊 `start_scheduler.bat`
2. 會開啟一個命令提示字元視窗
3. 最小化視窗即可（不要關閉）

**設定開機自動啟動：**

1. 按 `Win + R`，輸入 `shell:startup`，按 Enter
2. 在開啟的資料夾中，建立 `start_scheduler.bat` 的捷徑
3. 完成！下次開機時會自動啟動

## ⚙️ 修改排程時間

編輯 `apps/market_data/management/commands/start_scheduler.py`：

### 修改每日執行時間
```python
# 目前設定：週一到週五 13:30
scheduler.add_job(
    run_daily_tasks,
    trigger=CronTrigger(day_of_week='mon-fri', hour='13', minute='30'),
    #                                    ^^^^         ^^^^
    #                                    小時         分鐘
    ...
)
```

### 修改每週執行時間
```python
# 目前設定：週日 01:00
scheduler.add_job(
    run_weekly_recalc,
    trigger=CronTrigger(day_of_week='sun', hour='1', minute='0'),
    #                                    ^^^^       ^^^^
    #                                    小時       分鐘
    ...
)
```

### CronTrigger 參數說明
| 參數 | 說明 | 範例 |
|------|------|------|
| `day_of_week` | 星期幾 | `mon-fri`（週一到五）、`sun`（週日） |
| `hour` | 小時（24 小時制） | `13`（下午 1 點） |
| `minute` | 分鐘 | `30`（30 分） |

## 📊 查看執行紀錄

### 日誌檔案
所有執行紀錄會寫入：
```
logs/app.log
```

使用文字編輯器開啟即可查看詳細紀錄。

### 搜尋關鍵字
- `Starting daily tasks` - 每日任務開始
- `Stock crawl completed` - 股價爬取完成
- `Technical indicators completed` - 指標計算完成
- `Sector divergence completed` - 背離計算完成
- `ERROR` - 錯誤訊息

## 🛑 停止服務

**手動停止：**
1. 找到運行中的命令提示字元視窗
2. 按 `Ctrl + C`
3. 確認停止

**背景服務停止：**
1. 工作管理器 → 找到 `python.exe`
2. 結束工作

---

## ⚠️ 注意事項

1. **保持網路連線** - 排程任務需要網路才能爬取股價
2. **不要關閉終端機** - 關閉會停止排程服務
3. **定期檢查日誌** - 確保任務正常執行
4. **防火牆設定** - 允許 Python 訪問網路

---

## 🔧 故障排除

### 問題：任務沒有按時執行
**解決方法：**
1. 檢查排程服務是否運行中
2. 查看 `logs/app.log` 是否有錯誤訊息
3. 確認時間設定正確（注意時區是 Asia/Taipei）

### 問題：爬蟲失敗
**解決方法：**
1. 檢查網路連線
2. 查看證交所網站是否正常
3. 檢查日誌中的詳細錯誤訊息

### 問題：記憶體不足
**解決方法：**
1. 重新啟動排程服務
2. 考慮增加系統記憶體
3. 減少每週重算的頻率（例如改為每月）

## 💡 建議

1. **首次設定後**，觀察幾天確保任務正常執行
2. **每週重算** 會耗時較長（約 2-3 小時），建議在離峰時間執行
3. **定期備份** `db.sqlite3` 資料庫檔案
4. **長假前後** 手動檢查一次，確保沒有遺漏交易日
