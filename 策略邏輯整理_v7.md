| 項目 | 條件 | 性質 | 實作 |
|------|------|------|------|

## 一、第一階段：Signal Pool（標誌性動作股票池）

### 入池條件（每日檢查，符合任一即入池）

| 標誌性動作 | 定義 |
|-----------|------|
| 跳空缺口 | 今日最低價 > 昨日最高價 |
| 長紅 | 今日收盤價 vs 昨日收盤價，漲幅 > 7% |

### 出池規則

```
進池後第 21 個交易日（D = 20）當天強制移出
被移出的股票不再進行後續判斷（除非有新的標誌性動作重新入池）
```

### 狀態

- Signal Pool 是**獨立暫存區**
- 股票在池子內的每一天（D = 0 ~ 19），都會被重新檢查「情境 + R20」
- 同一支股票**可以多次入池**（每次標誌性動作都是獨立事件）
- **Buy Pool 每日重新計算**，非累積（今天符合就進，明天不符合就不在名單上）

---

## 二、第二階段：情境判斷（前置分類）

每日對 Signal Pool 內尚未出池的股票（D = 0 ~ 19）檢查：

| 情境 | 條件 | 結果 |
|------|------|------|
| **A** | EMA120 < EMA60 | ✅ 進入 R20 判斷 |
| **B** | EMA120 > EMA60 **且** EMA20 > EMA60 | ✅ 進入 R20 判斷 |
| 其他 | 不符合 A 或 B | ❌ 淘汰，不進行 R20 判斷 |

**說明：**
- 情境 A/B 只是**進入 R20 判斷的前置分類**
- 兩種情境共用同一個 R20 < 0.9 門檻
- R60、R120 在此策略中**完全移除**，不做判斷

---

## 三、第三階段：R20 判斷（核心）

### 定義

```
T = 判斷日（今天）
A = 入池日（標誌性動作出現那天）
D = T - A（已在池子裡待了幾個交易日，D >= 0）

R20 = Average( Close(A - (20 - D)) ～ Close(A - 1) ) / Close(T)
```

### 區間長度間長度變化（隨 D 增加而縮短）

| D | 起點 | 終點 | 區間天數 | 說明 |
|---|------|------|---------|------|
| 0 | A-20 | A-1 | 20 天 | 入池當天 |
| 1 | A-19 | A-1 | 19 天 | 入池第 2 天 |
| 2 | A-18 | A-1 | 18 天 | 入池第 3 天 |
| ... | ... | ... | ... | ... |
| 18 | A-2 | A-1 | 2 天 | 入池第 19 天 |
| 19 | A-1 | A-1 | 1 天 | 入池第 20 天 |

### 特殊處理

```
當 D >= 20：區間無法計算（起點 >= 終點），跳過該股票
當資料不足（起點超出歷史範圍）：跳過
只有在 0 <= D <= 19 且資料充足時，才計算 R20
```

### 門檻

```
R20 < 0.9（參數可調整）
```

### 核心洞察

- 終點永遠固定在 **A-1（入池前一天）**
- 隨著 D 增加，窗口從「過去 20 天平均」逐漸收縮到「入池前一天單日價格」
- 越晚判斷，「扣低價」參考期越短，門檻越嚴格
- R20 < 0.9 表示：**過去 20 天（或更短期間）的平均收盤價，比現在便宜 10% 以上**

## 四、第四階段：Buy Pool（潛在買入池）

### 進池條件（同時滿足）

1. 在 Signal Pool 內（尚未滿 21 天，即 D = 0 ~ 19）
2. 符合情境 A 或 B
3. **R20 < 0.9**

### 每日更新機制

```
Buy Pool 每日重新計算，不是累積的
→ 今天符合條件的股票進入 Buy Pool
→ 昨天在 Buy Pool 但今天不符合的，自動移除
→ 買入動作只看今天的 Buy Pool
```

### 同一股票多筆 Entry

- 如果同一股票在不同日期有多次標誌性動作，每次入池都是**獨立事件**
- 但在買入時，同一股票只保留**最新的 Entry**（如果同一天有多個，取最新的）

## 五、第五階段：族群 Buy Pool（可選 Filter）

基於標準 Buy Pool，增加族群背離條件：

### 條件

```
該股票所屬族群，在「最近 5 個日曆天內」（含 Today）有 is_orange = True
```

### is_orange 定義（族群背離系統）

- 族群乖離率排名「連續 2 天」排在全市場前 5 名以內
- 從第 2 天開始亮橘燈，直到掉出前 5 名

### 篩選邏輯

```python
for stock in 標準 Buy Pool:
    sector = stock_to_sector[stock.code]
    orange_dates = sector_orange_lookup[sector]  # set of dates
    
    has_orange = False
    for lb in range(0, 6):  # today, yesterday, ..., 5 days ago
        check_date = today - timedelta(days=lb)
        if check_date in orange_dates:
            has_orange = True
            break
    
    if has_orange:
        納入族群 Buy Pool
        記錄 days_since_orange = lb
```

## 六、Portfolio Simulation（進出場邏輯）

### 進場

```
每天開盤前：查看今天的 Buy Pool 名單
如果名單上的股票不在持倉中，以「當天收盤價」買入
持倉上限：無限制（或設定為 N 檔，策略未定義）
```

### 出場條件（依 Entry 時的 EMA 狀態分類）

#### 情境 1：Entry 時 EMA20 > EMA60（多頭排列，Trend-Up）

```
1. 停損：收盤價 < 進場價 × 0.8（-20%）→ exit_reason: stop_loss

2. 停利流程：
   a. 當 close >= 進場價 × 1.15，標記 triggered_15pct = True
   b. triggered 後，若 EMA20 < EMA60（均線反轉）→ exit_reason: profit_ema_reversal
```

#### 情境 2：Entry 時 EMA20 < EMA60（空頭排列，Trend-Down）

```
1. 停損：收盤價 < 進場價 × 0.8（-20%）→ exit_reason: stop_loss

2. 等待 crossover：
   a. 當 EMA20 > EMA60（空翻多），標記 crossed_over = True
   b. 若持有 >= 20 天且未 crossed_over → exit_reason: max_wait_crossover

3. Crossover 後停利流程：
   a. 當 crossed_over 且 close >= 進場價 × 1.15 → triggered_15pct = True
   b. triggered 後，若 EMA20 < EMA60（反轉）→ exit_reason: profit_after_crossover
```

#### 情境 3：期末持有

```
到模擬結束日（2026-05-08）仍持有的股票
→ exit_reason: end_of_period
→ 計算報酬率 = (期末收盤價 / 進場價 - 1) × 100%
```

### 狀態追蹤（每筆持倉）

```python
portfolio[stock_code] = {
    'entry_date': date,
    'entry_price': close,
    'entry_ema20': ema20,
    'entry_ema60': ema60,
    'triggered_15pct': False,  # 是否觸發 +15%
    'crossed_over': False,     # 是否 EMA20 上穿 EMA60（僅空頭進場時有意義）
    'days_held': 0,            # 已持有交易日數
    'scenario': 'A' or 'B',    # 進場時的情境
}
```
